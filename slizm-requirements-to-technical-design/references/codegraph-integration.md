本协议用于在已经建立 CodeGraph 索引的目标项目中优先取得语义上下文。CodeGraph 负责发现符号和关系；当前源码、配置、构建、测试与运行结果负责证明事实。

## 1. 适用条件与前置路由

读取用户材料、工作说明和非代码文档后，在首次代码搜索或源码读取前判断是否需要理解或定位代码。

需要完成以下任一工作时执行 CodeGraph 路由：

- 查找功能入口、类、方法、接口、配置或 schema；
- 理解调用链、动态分派、数据流、状态流或副作用；
- 查找接口实现、调用方、测试、复用点或影响范围；
- 判断模块、文件或符号职责；
- 修改代码前建立上下文，或修改后复查关键调用关系。

执行路由：

```text
需要理解或定位代码
→ 确认目标业务项目根目录
→ 检查该根目录的 .codegraph/
→ 存在：CodeGraph-first
→ 不存在：记录 not-indexed，使用普通取证
```

目标项目没有 `.codegraph/`、任务没有代码分析需求、或只读取已知位置的非代码资料时，不调用 CodeGraph。Skill 自身所在仓库的索引不能代替目标业务项目的索引。

## 2. 能力选择

只有目标项目根目录存在 `.codegraph/` 时，按以下顺序选择能力：

1. 当前宿主暴露标准 MCP 能力 `codegraph_explore` 时，优先使用它。
2. MCP 不可用或调用失败时，检查并使用 `codegraph explore` CLI。
3. CLI 也不可用或查询失败时，记录原因并使用普通搜索和直接读取。

不得为了满足工作流而自动执行 `codegraph init`、全量重建或索引修复。重新索引是完成任务的必要条件时，说明原因并由用户决定。

## 3. 探索查询

第一次查询覆盖完整语义路径，避免只搜索单个关键词：

```text
围绕需求或功能 <主题>，定位实际生效入口、核心调用链、接口与实现、
状态或数据来源、配置和外部依赖、输出副作用、现有测试以及主要影响范围。
返回具体文件、符号、行号和关系。
```

已知文件、类、方法或接口时把名称加入查询。结果不足时按问题窄化：

- 谁调用目标符号，目标又调用什么；
- 接口有哪些实际实现，动态分派可能到达哪里；
- 请求、状态或字段从哪里产生、写入、转换和持久化；
- 配置或 schema 在哪些路径生效；
- 哪些测试覆盖当前行为；
- 修改目标符号会影响哪些消费者；
- 当前是否已有可以复用的实现。

同一组新鲜结果可以被多个紧邻步骤复用。CodeGraph 已明确返回所需源码和关系后，不重复执行等价的全仓库搜索。

## 4. 一手证据核验

按返回内容选择核验方式：

- 返回目标符号的完整当前源码、定位和关系时，可把该源码作为读取证据；
- 只返回摘要、关系或不完整片段时，继续读取对应源文件和符号；
- 配置、生成源、模板、迁移文件、资源或未索引语言直接读取；
- 正式结论至少记录文件、符号或配置项、关系和证据边界；
- 模糊名称匹配、缺失调用者或单一摘要不能单独证明不存在其他路径。

CodeGraph 结果只证明代码现状，不是需求来源。现有行为不得自动升级为目标需求、保持不变项或验收标准。

## 5. 新鲜度与重新查询

在以下事件后重新确认相关结果：

- 切换分支，或 pull、merge、rebase 后；
- 大量文件移动、生成或删除后；
- 返回 stale、pending reindex 或等价状态；
- 查询结果与直接读取的源码不一致；
- 关键文件被修改，或用户及其他进程改动相关代码；
- 设计范围扩张、出现计划外模块或公共契约变化；
- 最终影响面审计前。

局部结果过期时，对相关文件直接读取并记录 `stale-partial`。整体同步异常时记录 `stale-global`，改用普通取证。不得用旧索引支撑当前结论。

## 6. 降级与状态记录

记录一个最终路由状态，并按需补充失败原因：

- `used-mcp`：使用 MCP 完成语义探索；
- `used-cli`：使用 CLI 完成语义探索；
- `not-indexed`：目标项目没有 `.codegraph/`；
- `mcp-unavailable`：MCP 不可用且后续路径也未完成语义探索；
- `cli-unavailable`：CLI 不可用且未完成 CodeGraph 探索；
- `query-failed`：可用能力查询失败；
- `stale-partial`：相关结果局部过期，已直接读取补充；
- `stale-global`：索引整体不可依赖；
- `not-applicable`：任务不需要代码探索。

降级后使用 `rg` 或等价文本搜索、直接读取关键文件、构建工具、类型检查、测试和必要运行验证。CodeGraph 不可用本身不阻断任务；只有缺少的语义证据会改变结论时才转为对应 Skill 的阻断状态。

## 7. 安全与完成门禁

始终遵守：

- 不执行 `codegraph init`，不修改或删除 `.codegraph/`；
- 不使用其他仓库的索引证明目标项目；
- 不把 CodeGraph 当成需求来源；
- 不用 CodeGraph 替代编译、测试、静态检查或运行验证；
- 不因未发现调用者就断言不存在动态调用；
- 不用无意义的重复扫描覆盖已经取得的充分语义证据。

完成代码事实分析前确认：应调用时已优先调用 CodeGraph；关键结论已落到当前文件、符号和关系；降级或新鲜度限制已披露；验证仍由项目工具独立完成。
